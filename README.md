# gravity-build

Плагин Claude Code: собирает прототипы и приложения **«как принято»** на настоящих компонентах
[Gravity UI](https://gravity-ui.com) — без выдуманных пропов, без «похожих» компонентов, с
правильным выбором библиотеки, app-shell-обвязкой и закреплёнными версиями.

> **Бета (v0.1.0).** Knowledge-слой обкатан на ~30 прогонах и реальном приложении; ставится через
> marketplace. Нашёл проблему или промах — см. [Фидбек](#фидбек).

## Что умеет

- **Идиоматичный код** — компоненты, пропы и версии сверяются с registry, а не берутся по памяти.
- **Правильная библиотека** — `@gravity-ui/uikit`, `navigation`, `table`, `page-constructor`,
  `icons`, `dynamic-forms`: выбор и обвязка под задачу (лендинг / дашборд / форма / настройки / …).
- **Per-service «как принято»** — опциональный профиль накладывает тему/бренд, плотность,
  раскладку, форм-политику и словарь конкретного сервиса.

## Скиллы

| Скилл | Что делает |
|---|---|
| `create` | Собирает UI: registry + (опц.) профиль → React + TypeScript. |
| `synthesize-profile` | Интервью → создаёт `service-profile.json` («как принято у нас»). |
| `write-back` | Сворачивает фидбек по сборке в нужный слой знания (профиль / registry / фикс). |
| `feedback` | Помогает тестеру оформить фидбек в структурированный отчёт + GitHub-issue. |

## Установка

Это **две отдельные команды** — введи первую, нажми Enter, дождись `✔`, затем вторую. Не вставляй обе разом.

Добавь marketplace:

```text
/plugin marketplace add https://github.com/ykamendrovskiy/gravity-build.git
```

Установи плагин:

```text
/plugin install gravity-build@gravity-build
```

Дальше просто попроси собрать что-нибудь на Gravity UI — скилл подхватится сам.

> **Падает с `git@github.com: Permission denied (publickey)`?** Это локальная SSH-настройка, а не репозиторий — он публичный. Полный `https://…`-URL в команде выше клонирует по HTTPS (авторизация не нужна), минуя SSH.

## Профиль сервиса (опционально)

Чтобы наложить вкус конкретного сервиса, положи `service-profile.json` в корень целевого проекта
(или `.gravity/service-profile.json`). Создать профиль через интервью — скилл `synthesize-profile`.
Без профиля сборка идёт по универсальным дефолтам.

## Фидбек

Это бета — фидбек ценен. Два пути:

- Запусти скилл **`/gravity-build:feedback`** — он соберёт структурированный отчёт и даст
  пред-заполненный issue.
- Или открой [issue вручную](https://github.com/ykamendrovskiy/gravity-build/issues/new?labels=beta-feedback).

> Репозиторий публичный — не вставляй в отчёты внутренние имена сервисов, URL и доменные данные.
