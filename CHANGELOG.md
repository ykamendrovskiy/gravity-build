# Changelog

## 0.2.6 — beta (2026-07-03)

Итоги параллельного прогона трёх прототипов (лендинг / админка / мастер) с автоматическими проверками и разбором вживую. Способ использования не меняется.

- **Ширины колонок таблицы — рабочая модель.** Три класса колонок: тянущиеся (название, адрес — забирают свободную ширину), статичные (дата, сумма — точная ширина) и служебные (чекбоксы выбора, действия «⋯» — не растут никогда). Механика: `table-layout: fixed` + `width: auto` колонке-поглотителю + `min-width` самой таблице — без него при сужении окна тянущаяся колонка схлопывается в ноль. Дополнительно: у таблицы `@gravity-ui/table` нет собственного контейнера прокрутки — горизонтальный скролл делается обёрткой вокруг самой таблицы (не вокруг панели действий).
- **Отзывы на лендинге** — карточка `basic-card`: имя заголовком, цитата текстом, роль отдельной подписью (`additionalInfo`), аватар — маленькой картинкой в `icon`-слоте. Прежний вариант `quote` с большой боковой картинкой тесен даже в две колонки (колонка подписи рвётся по словам); `quote` остаётся для случаев с настоящими фотографиями. Важно: markdown внутри `text` карточек не обрабатывается — подпись кладётся в свой слот, не разметкой. Заодно исправлена старая формулировка о выделении тарифа, противоречившая уточнению версии 0.2.2.
- **Экран успеха / результата** — через `PlaceholderContainer` с иллюстрацией (вертикальная ориентация, `direction="column"`), а не самодельный центрированный блок с иконкой: самодельный вариант оставляет иллюстрации и их покраску неиспользованными.
- **Кнопки без текста** оборачиваются в `ActionTooltip` с полным названием действия (поддерживает описание и хоткей); кнопкам с текстом подсказка добавляется по сложности действия.
- **Индикатор шагов (`Stepper`):** `id` шагов и `value` передаются строками, оба сразу — числовой `id` не проходит типы, а числовой `value` при строковых `id` молча гасит подсветку текущего шага.

## 0.2.5 — beta (2026-07-02)

Три дополнения knowledge-слоя: точный маршрут до пропов любого компонента, достижимые состояния экрана с их декларацией, правило ширины таблицы на страницах-списках. Способ использования не меняется.

- **Точные пропы без угадывания.** Новый справочный маршрут: если нужного пропа или компонента нет в гайдах, читаются типы установленного пакета (`package.json → types`, `.d.ts` рядом с компонентом — таблица путей для всех библиотек набора) и README компонента в репозитории на теге закреплённой версии. Важно: структура пакетов неоднородна (`dist/` у date-components, плоский корень у icons/illustrations, своя глубина у dynamic-forms) — теперь это записано, а не угадывается; при недоступной сети типы пакета остаются полным офлайн-ответом.
- **Состояния экрана достижимы и декларируются.** Прототип со списком/формой строит все состояния (загрузка / пусто / ошибка / «ничего не найдено») через реальный поток данных и объявляет их в `scenarios.manifest.json`. Переключатель `?scenario=` работает в dev и в сборке с `VITE_SCENARIOS=1`; в обычную прод-сборку он не попадает. Декларация позволяет автоматически проверять каждое состояние, а не только начальный экран.
- **Таблица на странице-списке заполняет контент-колонку** (`width="max"`): действие в шапке, фильтры и таблица выравниваются по одному правому краю — кнопка больше не «повисает» в пустоте правее таблицы (у uikit-таблицы ширина по умолчанию сжимается под контент). Другие раскладки остаются осознанным выбором сервиса.

## 0.2.4 — beta (2026-06-30)

Разбор промахов раскладки и вида из живого прогона в браузере (лендинг / дашборд / визард). Способ использования не меняется.

- **Отзывы и другие карточки с крупной картинкой сбоку.** Правило уточнено: проблема только у карточек с большой боковой картинкой — их выводим 1–2 в ряд (слайдер или сетка) ЛИБО перестраиваем (картинка сверху / компактно) для 3 в ряд. Плюс рабочие детали слайдера: нужен `import 'swiper/css'` (и в билде, не только в dev), а число видимых карточек задаётся **числом** (`slidesToShow: 2`), не ключами `{desktop, tablet}`.
- **Кнопки на тёмных блоках лендинга.** Вторичная кнопка на тёмном блоке теперь читаема — используются `-contrast`-темы (`outlined-contrast`), иначе тёмный текст на тёмном фоне.
- **Подтверждение удаления — единым способом.** Удаление и массовое, и построчное идёт через один `ConfirmDialog`; браузерный `alert`/`confirm` не используется, отклик действий — тост.
- **Иллюстрации состояний видимы везде.** `PlaceholderContainer` использует иллюстрацию (не растянутую иконку), включая экран успеха; покраска `--gil-color` обязательна, иначе иллюстрация невидима.
- **Кнопка в ряду с полями — того же размера** (не выбивается по высоте).
- **Индикатор шагов.** Отметка «выполнено» хранится в состоянии (не теряется при возврате назад); переход по клику на шаг — при явной настройке.

## 0.2.3 — beta (2026-06-30)

Ещё несколько промахов раскладки и вида из живого прогона в браузере. Способ использования не меняется.

- **Отзывы и другие карточки с картинкой сбоку от текста.** Их больше не ставим по 3 в ряд — у границы адаптива колонка текста схлопывается до нечитаемой. Рекомендация: 2 в ряд или слайдер по одной.
- **Плейсхолдеры-логотипы в цитатах.** Текст внутри инлайн-SVG экранируется как XML (`&`→`&amp;`) — иначе название с `&` (напр. «Studio&Co») ломает картинку.
- **Индикатор шагов (`Stepper`).** Он рендерится как список (`<ol>`) и наследует браузерный отступ слева; в reset добавлена нормализация списков (`ul, ol { margin: 0; padding: 0 }`), чтобы степпер вставал по сетке, а не уезжал вправо.

## 0.2.2 — beta (2026-06-30)

Живой прогон лендинга и приложений в браузере вскрыл несколько промахов раскладки и вида — исправлены в knowledge-слое. Способ использования не меняется.

- **Выделение тарифа (`page-constructor`) — исправление формулировки 0.2.1.** Тема карточки красит только текст; чтобы выделить план, нужен либо тёмный фон (`backgroundColor`) **вместе** с тёмной темой, либо рамка (`border`). Прежняя формулировка «выделяй темой, не фоном» приводила к невидимому тексту (белый на прозрачном).
- **Сетка лендинга.** В глобальный reset добавлен `box-sizing: border-box` — без него сетка `page-constructor` даёт горизонтальный скролл на десктопе, а карточки встают 2-в-ряд вместо 3.
- **Иконка в кнопке действия.** Иконку с текстом передавать массивом детей, а не фрагментом — иначе иконка наезжает на текст (собирается без ошибок, но видно только в браузере).
- **Таблица с действиями (`@gravity-ui/table`).** У её `Table` нет пропов `width`/`stickyHeader` (это компоненты uikit `DataTable`) — полная ширина через `attributes`; уточнена сигнатура хелпера колонки действий.
- **Плейсхолдеры-картинки в цитатах.** Инлайн-SVG (аватар / логотип) кодировать через `encodeURIComponent`, а не `btoa` — иначе кириллица роняет страницу в пустоту.
- **Согласованность вида.** Вставленный компонент (например, редактор текста) по виду синхронизируется с соседями (рамка), если сам лэйаут их не разделяет.

## 0.2.1 — beta (2026-06-30)

Уточнения knowledge-слоя по сверке с исходниками Gravity UI (`uikit` 7.42, `page-constructor` 8.13, `date-components` 3.4). Способ использования не меняется.

- **Иконки в контролах:** правильные слоты (`Button` — children, `TextInput` — `startContent`, `Label` — `icon`); размер задаётся вручную (uikit не масштабирует svg); **отступ ведущей иконки** в поле подбирается под ритм текста (зависит от размера иконки и контрола).
- **Массовые действия (`ActionsPanel`):** выверенная раскладка — ширина совпадает с таблицей (`width="max"`), поштучные действия строки сосуществуют с массовыми, панель закрепляется у нижнего края (по умолчанию — на всю страницу); тип элемента — `ActionsPanelProps['actions']` (`ActionsPanelItem` не экспортируется).
- **Таблица:** типизация цепочки table-HOC без приведения типов; выравнивание колонки — логические `start`/`end` (физические `left`/`right` устарели).
- **Формы:** валидация задаётся на самом контроле (`validationState` / `errorMessage`) — единообразно для всех полей ввода, включая `@gravity-ui/date-components`.
- **Темизация:** тёмная секция внутри светлой страницы — через `<ThemeProvider scoped>`.
- **Лендинги (`page-constructor`):** выбор «слайдер против статичной сетки `card-layout-block`», выделение тарифа темой (а не фоном — иначе текст пропадает), полный список под-блоков; у цитаты обязательны `image` и `logo`.
- **Прочее:** `index.html` в корне проекта (для Vite).

## 0.2.0 — beta (2026-06-26)

- **Внутренняя реорганизация knowledge-слоя** в двухслойную модель: единый роутер-указатель + справочники. Факт описан в одном месте, версии собраны вместе — меньше расхождений и дрейфа. **Способ использования не меняется** — те же скиллы, та же установка.
- Рецепты стали **тоньше**: дают специфику Gravity и неочевидные части сборки, а композицию страницы целиком модель собирает по описанию вида — так пример не копируется буквально вместе с огрехами.
- Подтверждение необратимых действий и закрытие формы с несохранёнными данными — через `ConfirmDialog` (`@gravity-ui/components`); добавлен его точный набор пропов (шапка — `title`, не `caption`).
- Массовые действия над выбором строк (`ActionsPanel`) вынесены в отдельный справочник + точная форма элемента действия.
- Состояния пустого списка и ошибки в дашборде — иллюстрации подключены к набору пакетов рецепта.
- Уточнения по типам: валидация поля задаётся на самом контроле (`validationState` / `errorMessage`), не на обёртке `FormRow`; для просмотра состояний в DEV (`import.meta.env`) нужен `src/vite-env.d.ts` (иначе ошибка типов).
