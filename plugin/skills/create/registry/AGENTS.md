# Registry — как driver'у работать

Knowledge-слой для сборки прототипов на настоящих компонентах Gravity UI «как принято».

**Entry-point — `registry.json` (роутер).** Прочти его первым и следуй полю `workflow`: scaffold → pattern →
interface-foundations → recipe → gravity-foundations → libraries. Маршруты / версии / факты — в роутере и в
`guides/`. Не подставляй имена / пропы / версии по памяти других дизайн-систем — сверяй с guide библиотеки
(особенно «Ловушки» / «Грабли пропов» в `guides/library-uikit.md`) **перед** импортом.

**Вход — Figma-макет** (ссылка/fileKey/node-id)? Перед всем остальным — `guides/figma-mapping.md`
(source-intake: протокол чтения канала, карта имён кита, политика фиделити «макет = что строить,
registry = как»); выход переноса включает конфликт-отчёт.

**Если прямой фетч upstream заблокирован** (WebFetch / gh недоступны) — **первый офлайн-путь: типы уже
установленного пакета** (`package.json → types`, co-located `.d.ts` — маршрут в `guides/reference-props.md`);
дальше — **WebSearch** по README / AGENTS библиотеки. Только если и это не вышло — собери из проверенных
registry-примитивов и **пометь комментарием**; НЕ хардкодь компонент / пропы по памяти от безысходности (даже
если кажется, что компонент такой есть — напр. `Stepper` существует, но его API надо проверить, а не сочинить).

## Профиль сервиса (если передан)

Если вместе с registry дан **профиль сервиса** (`service-profile.json` — отдельный файл, НЕ часть registry), он
задаёт «как принято именно в этом сервисе»: тема / бренд, плотность, раскладка контента, форм-политика, сабсет
компонентов, архетипы, доменный словарь. Применяй:

- **Профиль уточняет открытые ручки**, которые гайды оставили на дефолте (плотность, выравнивание / max-width
  контента, ширина полей формы и размещение описаний, тема) — по каждой бери значение из профиля вместо дефолта.
- **Профиль НЕ отменяет корректность.** Имена компонентов, версии, app-shell-обвязка, «FormRow вместо
  инлайн-`label=`» — это registry; при конфликте директивы профиля с корректностью побеждает registry, пометь комментарием.
- **Чего профиль не задал — бери дефолт гайда** (омиссия = дефолт; профиль может быть тонким).
- `approvedComponents`: `deny` — не использовать; `preferred` — разрешает универсальные «it depends»-развилки
  (напр. «точное число → NumberInput»).
- **Тема / бренд:** `theme.themeConfig` → следуй `guides/gravity-foundations-theming.md` (скопируй указанный
  `.theme.css` в проект, импортируй **после** uikit `styles.css`, токены руками не подбирай); `theme.brandTokens`
  → полный набор per-theme; ничего → дефолтная тема Гравити.

Если профиль **НЕ передан** — игнорируй этот раздел, собирай по универсальным дефолтам.
