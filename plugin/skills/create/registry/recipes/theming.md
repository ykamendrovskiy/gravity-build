# Pattern: Темизация / брендирование

Тема подключается в app-shell (`ThemeProvider` + `styles.css`, см. `pattern-app-shell`). **Бренд / кастомная тема** — отдельный слой поверх неё. Открывай этот гайд, когда у сервиса есть свой бренд или кастомная тема.

## Почему бренд — это набор, а не один токен

Цвета Гравити двухслойные: **приватная палитра** (оттенки 50–1000) → **семантические токены** (`--g-color-base-brand`, `--g-color-text-brand`, `--g-color-line-brand`, `--g-color-base-selection`, `--g-color-line-focus`, `--g-color-text-link*` …). Компоненты используют семантические. Поэтому сменить бренд = задать **полный набор** связанных токенов, причём:

- **на обе темы** (light и dark — **разные** значения: в dark бренд обычно ярче);
- обязательно **`--g-color-text-brand-contrast`** (текст/иконки поверх brand-фона) — иначе кнопка может стать нечитаемой; проверь контраст во всех темах.

Переопределить «2–4 токена» (`--g-color-base-brand` и пару) — типичная ошибка: selection / focus / ссылки / контраст останутся дефолтными → рассогласование.

## Три способа (бери первый доступный)

### 1. Themer-тема (профиль ссылается на неё) — основной путь

Если в `service-profile` есть `theme.themeConfig` — тема уже сгенерирована (Themer посчитал всю палитру + семантику на все темы). Твоё дело — **подключить файл**, токены руками не трогать:

```tsx
// src/main.tsx
import '@gravity-ui/uikit/styles/fonts.css';
import '@gravity-ui/uikit/styles/styles.css';
import './theme.css';   // ← .theme.css из профиля, скопированный в проект. ВСЕГДА после styles.css
```

`.theme.css` содержит блоки `.g-root_theme_light { … }` / `.g-root_theme_dark { … }`. `ThemeProvider theme="light|dark"` ставит класс `g-root_theme_<тема>` → нужный блок подхватывается автоматически. Ничего больше не нужно.

### 2. brandTokens (есть бренд-цвет, нет Themer-темы) — fallback

Если `theme.brandTokens` (или просто известен бренд-цвет): задай **полный** набор brand-токенов, **раздельно по темам**, scoped в `.g-root_theme_light/_dark`, с `--g-color-text-brand-contrast`. Лучше — всё же сгенерировать тему (см. ниже) и вернуться к способу 1.

### 3. Ничего — дефолтная тема Гравити

Нет брендового слоя → не переопределяй ничего. Омиссия = дефолтный акцент Гравити.

## Создать или отредактировать тему

- **[Themer](https://gravity-ui.com/themer)** — визуальный конструктор; кнопка **Export** даёт JSON (source-of-truth). Тот же JSON заливается в Figma (плагин Gravity Themer) → дизайн и код на одной теме.
- **`@gravity-ui/uikit-themer`** — программная генерация палитры из базового цвета.

JSON — источник правды; CSS (`.g-root_theme_*`) — производная для импорта. Ребренд = заменить JSON → перегенерить CSS, **не** править `--g-*` по всему коду.

## Контролы на брендовом / цветном фоне — контраст

Когда объект (hero-блок, баннер, карточка с акцентным фоном) залит **брендовым/цветным фоном**, контролы на нём должны **читаться против этого фона**, а не следовать теме. Типичная ошибка: дефолтная кнопка на брендовом фоне сливается или даёт низкий контраст (напр. светлая кнопка на ярком фоне).

- **Фиксированный контраст, обе темы.** Брендовый фон обычно **одинаков в light и dark** (это не тематическая поверхность, а заливка). Значит, и контрол на нём должен иметь **фиксированный** контрастный вид в обеих темах (напр. тёмный), а не светлеть/темнеть вместе с темой. Не полагайся на дефолтный `view`, который следует теме.
- **Как задать (по порядку):** (1) выбери `view` с собственной контрастной поверхностью (напр. `raised` — приподнятый солид, читается на цветном лучше прозрачных `flat`/`outlined`); (2) если подходящего `view` нет — задай цвета через **CSS API** компонента (`--g-button-background-color`, `--g-button-text-color` и т.д., scoped на обёртку блока), зафиксировав контрастные значения на обе темы; (3) проверь контраст текста/иконок (WCAG) на фактическом фоне.
- Не путать с `--g-color-text-brand-contrast` — тот про текст **на** brand-кнопке (`view="action"`); здесь — про контрол **на** brand-**фоне**.

## Чего НЕ делать (anti-patterns)

- Переопределять 2–4 токена — selection / focus / ссылки / `text-brand-contrast` останутся дефолтными.
- Одно значение бренда на light и dark — в dark нужно ярче, контраст другой.
- Забыть `--g-color-text-brand-contrast` — риск нечитаемого текста на brand-кнопке.
- Импортировать тему **до** `styles.css` — uikit перезапишет переопределения (custom всегда **после** `styles.css`).
- При ребренде искать и править `--g-*` по всему коду вместо одной темы-файла.
