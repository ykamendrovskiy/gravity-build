# pattern-list — список сущностей (агностичный спек вида)

> Тип: `pattern`. **Что есть хорошая страница-список** (вид интерфейса), безотносительно конкретной сборки.
> Композиция + как ЭТОТ вид реализует состояния. **Ссылается** на `interface-foundations` (дисциплина состояний,
> data-safety, feedback) — не повторяет их. Гравити-сборка — в `recipe-list`.

## Композиция (что на экране)

1. **Заголовок** страницы + основное действие («Добавить …») справа.
2. **Фильтры** над таблицей (поиск + категориальные фильтры).
3. **Таблица** записей (колонки по домену; статусы — метками).
4. **Выбор строк** ⇒ панель массовых действий (если есть bulk).
5. **Добавление** — диалог с формой.

## Как список реализует состояния (контракт → см. interface-foundations §1)

- **Loading:** `Skeleton` на месте **строк** таблицы (block-level), раскладка не прыгает.
- **Empty:** `PlaceholderContainer` (иллюстрация + заголовок + текст + действие). Различай подтипы:
  *пусто впервые* («Добавить оборудование») vs *нет результатов фильтра* («ничего не найдено» + «сбросить фильтры»).
- **Partial:** рендер что есть + пагинация/«показать ещё»; фоновый refetch — вуаль поверх таблицы.
- **Error (page-level):** `PlaceholderContainer` + «Обновить» (НЕ inline `Alert`).
- **Ideal:** таблица с данными.

## Мутации (добавление/правка) → feedback + data-safety

- Сабмит — видимый pending/error (interface-foundations §3).
- **Удаление строки/балк → подтверждение `Dialog`** (interface-foundations §2).
- Диалог добавления с введёнными данными **не закрывается** по outside-click/Esc без подтверждения.

## Ручки сервиса (profile, если передан)

`density` (плотность строк), `filtersPlacement` (над таблицей / в сайдбаре), `emptyCopy` (тон), `retryPolicy`
(авто/ручной). Омиссия = дефолт.
