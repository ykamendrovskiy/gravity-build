# interface-foundations — агностичные UX-базы (сквозные по паттернам)

> Тип: `interface-foundations`. **Агностично** (не про Гравити-тулкит, а про то, что делает ЛЮБОЙ интерфейс
> продакшен-готовым). Паттерны **ссылаются** сюда — не дублируют. Граница: *какие* состояния/правила
> существуют — здесь (универсально); *что выбирает сервис* (тон копирайта, page vs block, авто/ручной retry)
> — в профиле.

## 1. Дисциплина состояний (UI Stack)

Любой экран с **асинхронными** данными обязан обработать **все 5 состояний**, не только happy-path. Источник
модели — UI Stack (Scott Hurff). Состояния должны быть **реализованы и достижимы** (не мёртвый код) и в проде
возникать из **реального потока данных**: loading = идёт реальный async-запрос; empty = данных правда нет;
error = реальный сбой. **НЕ ставь в продакшен-UI переключатель состояний** — пользователю он бессмыслен.
Чтобы увидеть состояния при разработке — отдельный dev-харнесс (см. «Состояния в DEV» ниже).

| Состояние | Что | Общий Gravity-маппинг |
|---|---|---|
| **Loading** | первый показ, данные грузятся | `Skeleton` (держит раскладку, без сдвига), НЕ одинокий `Spin` по центру |
| **Empty** | запрос ок, данных нет (3 подтипа: онбординг / очищено / нет результатов фильтра) | `PlaceholderContainer` (иллюстрация+текст+действие) |
| **Partial** | данные есть, но неполны | рендер что есть + аффорданс дозагрузки; при фоновом refetch — вуаль поверх (без прыжка) |
| **Error** | запрос упал | **page-level** load-fail = `PlaceholderContainer` + retry (full-area, как Empty); **block/фон** = компактно (тост). `Alert` — только in-context system/domain (обслуживание/квота), НЕ техсбой загрузки |
| **Ideal** | happy-path | обычный рендер |

Гранулярность: **page-level** (1 лоадер/ошибка на страницу) vs **block-level** (по-блочно). На списках — скелетон
**строк** (block), не один лоадер на всю страницу. Конкретную реализацию состояния для своего вида — см. свой `pattern`.

## Состояния в DEV — как смотреть (НЕ в проде)

Состояния должны быть видимы разработчику, но **не** контролом в самом продукте. Подходы по весу:

1. **Dev-gated переключатель / URL-параметр — дефолт для прототипа.** Сценарный свитчер или `?scenario=empty`,
   но **только под dev-флагом** `import.meta.env.DEV` (Vite) → в `vite build` tree-shake'нется, в прод не попадёт:
   ```tsx
   {import.meta.env.DEV && <ScenarioSwitcher value={scenario} onChange={setScenario} />}
   // или: const scenario = import.meta.env.DEV
   //   ? (new URLSearchParams(location.search).get('scenario') ?? 'ideal') : 'ideal';
   ```
   Смотришь: `npm run dev` (+ `?scenario=empty`). Дёшево, видно сразу, в проде нет.
   **NB:** `import.meta.env` требует `vite/client`-типы — добавь `src/vite-env.d.ts` c `/// <reference types="vite/client" />`, иначе `tsc` → **TS2339** (см. `scaffold-app-shell`).
2. **MSW (mock service worker) — когда нужна реалистичность.** Компонент делает **настоящий** `fetch`;
   MSW-сценарии отдают success / empty / 500 / задержку. Состояния возникают из реального data-flow (как в
   проде) — переключаешь мок, не код. Честнее всего, но это service-worker-инфра.
3. **Storybook — для библиотеки компонентов.** Каждое состояние = story; идиоматично для Гравити, но оверкилл
   для одиночного прототипа.

**Правило:** прод — состояния из реального потока данных; видимость в деве — способ 1 по умолчанию.

## 2. Безопасность данных (data-safety)

Сквозная политика — бьёт любой деструктив/ввод, не привязана к одному виду экрана:

- **Деструктив → подтверждение.** Удаление (строки / балк / необратимое) — через диалог подтверждения, не молча (Гравити: `ConfirmDialog` — см. `library-components`).
- **Несохранённый ввод → защита от потери.** Диалог/форма с введёнными данными НЕ закрывается по
  outside-click / `Esc` без подтверждения («Закрыть без сохранения?»). Пустую/нетронутую — можно закрывать.
- Достижимость: применяй при ключевых словах `delete` / `destructive` / `close` / `unsaved`.

## 3. Feedback (видимая реакция)

- **Submit-pending:** кнопка отправки `disabled` + `Loader`, пока идёт.
- **Submit-error:** видимое сообщение, форма **не** закрывается.
- Никаких «тихих» `console.log` вместо реакции. Optimistic (строка сразу, откат при ошибке) — опц.

## Что отсюда берёт сервис (profile)

Тон копирайта пустого, page vs block, skeleton vs вуаль, авто vs ручной retry — выбирает профиль (если передан).
Омиссия = разумный дефолт. Сами правила (все 5 состояний; деструктив→confirm; видимый feedback) — обязательны.
