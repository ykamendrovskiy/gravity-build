# figma-mapping — перенос Figma-макета в Gravity-сборку (source-intake)

> Тип: route-guide, семейство source-intake (ADR-0006). Открывай, когда **вход сборки — Figma-макет**
> (ссылка на файл/фрейм, fileKey/node-id). Направление истины: **макет = ЧТО строить** (контент verbatim /
> раскладка / exercised-тема), **registry = КАК строить** (имена / идиомы / версии / app-shell). Не наоборот:
> макет не отменяет корректность registry, registry не «улучшает» контент макета молча.

## Режимы канала (объяви режим в отчёте переноса)

1. **Полный MCP (дефолт):** `get_metadata` / `get_design_context` / `get_variable_defs` / `get_screenshot` /
   `download_assets`; иконки приходят Code-Connect-кодом (готовый импорт `@gravity-ui/icons` — доверенный слой).
2. **REST-PAT-адаптер:** MCP нет, есть Figma PAT (scope `file_content:read`) — `GET /v1/files/<key>/nodes`.
   Прототип-реакции (стрелки переходов) MCP **не отдаёт вовсе** — они достижимы только здесь (`interactions` /
   `transitionNodeID`) либо конвенцией текст-подписей в макете.
3. **Скриншот+ручной (деградация):** теряет имена компонентов и вариант-пропы (главный источник «вёрстки на
   дивах»), точные токены (альфа-тона со скрина пипеткой не различить), verbatim-тексты и ассеты. Допустим
   только с явной пометкой «деградационный режим» в отчёте.

## Протокол чтения макета (порядок обязателен; бюджет ≈25k токенов на экран 1200×800)

1. `get_screenshot` фрейма → **эталон 1×** (URL → curl на диск). Дальше на всех `get_design_context`
   ставь `excludeScreenshot=true`.
2. `get_metadata` фрейма → структура + **все тексты** (кроме текстов внутри инстансов) + координаты.
   **Высоты зон и отступы считай из координат metadata, не на глаз.**
3. `get_design_context` — **только по секциям** (~⅓ экрана за вызов; целый фрейм канал не отдаёт).
   Повторяющийся блок (карточки списка) = design_context **одного** экземпляра, тексты остальных — из metadata.
4. Тексты **внутри инстансов** (подписи `Label`, сегменты контролов) metadata не несёт — только design_context.
5. `get_variable_defs` → токены **текущего мода** (dark не приходит). «В defs есть» ≠ «на экране используется»,
   но и «на глаз не видно» ≠ «не используется»: **тонкие тона проверяй пипеткой/сверкой токена, не глазомером**
   (кейс: base-background `#edf5f3` неотличим от белого на глаз, а тонирует весь холст).
6. Настоящие картинки инстансов — **только** `download_assets(instanceNode)` → `rawImages`;
   design_context инлайнит **дефолт-ассеты компонента** (плейсхолдеры кита). Обязательный шаг при фото/аватарах.

## Тема из макета (чек-лист exercised-слоёв)

- Переноси **все** exercised-слои variable_defs: **цвет + радиусы + типографика-скейл** — только цвета =
  «стилизация не полностью» (тема Гравити — это и радиусы тоже).
- **Семантический слой целиком, не выборочно.** Тема кита переопределяет не только бренд/фоны, но и
  **семантические токены** — они красят состояния контролов: прецедент S3 — розовый бордер чекбоксов =
  `--g-color-line-generic-accent: #ca155199` (uikit Checkbox красится именно им). Из variable_defs бери
  verbatim ВЕСЬ semantic-набор (`line-*`, `text-*`, `base-*-light`, selection) — «не заметил на экране» ≠
  «не используется»: композиты сходятся байт-в-байт только при полном оверлее (verified S3: чекбокс,
  Alert=misc-light⊕фон, яма=generic⊕фон, selection-строка). Контрол-состояния (unchecked-бордер
  checkbox/radio, selection-тона) сверяй **пипеткой по эталону** — класс «пипетка, не глазомер» ловил
  уже дважды (S1 фон-тонировка, S3 чекбокс).
- Шрифт кита (Inter) **не переносим** — шрифт = слой темы/профиля, не макета.
- Extraction = **частичная тема** (exercised-токены одного мода; hover / dark / focus канал не отдаёт) —
  честный floor. Полный бренд достраивается формулой **«generate → overlay»** (verified пробой S2):
  Themer-генерация из seed = base-brand **плюс явный base-background из extraction** (тонированные нейтралы
  тогда совпадают с китом байт-в-байт; бренд-производные — с дрейфом ≤4/канал), поверх — extraction-значения
  verbatim (макет = истина на пересечении). **Только extraction** (Themer не выводит): радиусы и структурные
  оверрайды кита (selection-оттенок, акцент-линии). Маршрут и механика — `gravity-foundations-theming`.
- Битые переменные кита (вида `--var(--g-text-body-font-weigh)t`) → конфликт-отчёт (баг кита), значение
  бери из фактического рендера.

## Карта имён: кит → код

| Кит-инстанс (признаки) | Gravity-компонент | Заметки |
|---|---|---|
| `Select` | `Select` (+`width="max"` при полноширинном) | size по высоте (шкала ниже) |
| `List-item` (Basic) | `unstable_ListItemView` | импорт из `/unstable` + рендер-грабли selected/height/divider — `library-uikit` |
| `List-item` (Divider) | `<li role="separator">` | divider-типа у ListItemView нет — `library-uikit` |
| `Cards` | `Card type="selection"` | selected-рендер ≠ кит — конфликт-класс «идиома», см. политику ниже |
| `Button` | `Button` | view/size — инференсом по стилям (ниже); icon-only ⇒ `ActionTooltip` (`library-uikit`) |
| сплит-кнопка (иконка+шеврон) | **одна** `Button`, две `Icon` массивом прямых детей | канон «иконка+текст/иконка» — `library-uikit` |
| `SegmentedRadioGroup` | `SegmentedRadioGroup` + `.Option` | selected-плашка рендерится по-своему — минор, оставляй штатной |
| `TextInput` (+иконка, +label) | `TextInput label placeholder startContent` | инсет и цвет старт-иконки — `library-icons` |
| `Label` (серый чип) | `Label theme="unknown"` | тексты чипов — только из design_context |
| `User` | `User avatar={{imgUrl}} name description` | аватар — через `download_assets`, не инлайн-плейсхолдер |
| `Switch` (+подпись) | `Switch content` | |
| иконки (CC-код) | `@gravity-ui/icons` — импорт из CC как есть | доверенный слой канала; имена не переизобретай |
| текст со стилем `--g-text-*` | `Text variant` (body-1 / body-short / subheader-*…) + `color` | скейл — `gravity-foundations-typography` |
| композер с wysiwyg-тулбаром | `@gravity-ui/markdown-editor` (ставь **bundle** из роутера) | НЕ самодельный тулбар из Button+TextArea; см. «Reply/редактор-зоны» |

Незамапленный инстанс / detach / кит-имя не из таблицы → **строка конфликт-отчёта, не угадывание** (для
пропов существующего компонента — `reference-props`).

**Контейнер без кит-инстанса ≠ ближайший смысловой kit-контейнер.** Карта имён — про ИНСТАНСЫ. Если
контейнерная зона макета собрана из plain-фреймов (кит-инстансы только у элементов внутри) — это
**кастомный компонент макета**: переноси его кастом-раскладкой (flex-колонка/грид) + кит-компоненты для
элементов, а НЕ ближайшим контейнер-компонентом с собственным поведением — тот привнесёт свои икон-слоты,
коллапсы и разделители и потеряет поведение макета. Прецедент S3: сайдбар из `List-item`-инстансов в
plain-фреймах ≠ `AsideHeader` (натяжка дала пустоту на месте икон-слотов у текстовых пунктов, потерю
шеврон-коллапса секций, лишний разделитель, увод Teams в More-коллапс) → кастом-`<aside>` +
`Text`/`Icon`/`Avatar`/`Button`-элементы, поведение секций по макету.

## Инференс вариант-пропов (канал отдаёт их неоднородно)

Часть компонентов несёт пропы именем ноды (`Alert`: `Theme=Warning, View=Filled`), часть — нет (`Button`).
Тогда — инференс по стилям/геометрии (устойчив, подтверждён сборкой):

- **px→size:** высота 20=`xs` · 28=`m` · 36=`l` (Button / TextInput / Select); `Label` 20=`xs`;
  `Icon` 16 в m/l-контролах, 12 у шеврона сплит-кнопки.
- **Button view по стилям:** фон base-brand → `action`; без фона → `flat`; бордер → `outlined`.

## Политика фиделити (кто прав)

**Дефолт: контент + раскладка + exercised-тема = макет; идиоматика кода = registry.**

- **Из макета verbatim:** тексты — включая опечатки, дубли и **кит-дефолт-протечки** (несмененные
  плейсхолдеры инстансов: «Label:», «UserName», «Content» на кнопке) — protечки помечай флагом в отчёте;
  normalize — только явной опцией заказчика, **silent-fix запрещён** (фиделити должна быть измерима).
  Размеры/пропорции колонок, плотность строк (`height` — с пометкой «плотность → профиль»), extracted-токены.
- **Из registry** (макет этого не показывает): scaffold/app-shell целиком, `ActionTooltip` на icon-only,
  инсет старт-иконок, overflow-канон непереносящихся тулбаров, контракт состояний + манифест сценариев
  (если строишь живой прототип, а не статичный перенос).
- **Конфликт-классы и дефолт-решения:**
  - **имя** — кит-имя ↔ код-имя расходятся → карта выше; нет в карте → отчёт.
  - **идиома** — компонент-дефолт рисует иначе, чем кит (Card selected: рамка vs заливка; ListItemView:
    чекмарк vs заливка): сперва **штатные пропы**; только затем CSS-надстройка **токеном** с пометкой-хаком;
    расхождение фиксируй в отчёте — системное «кто прав» = ручка профиля. **Owner-ruling S3: точное
    воспроизведение макета важнее дефолта компонента** — отход кита от дефолта может быть осознанным
    решением дизайнера; «у компонента так по умолчанию» — не аргумент оставить как есть (прецедент:
    DefinitionList без точечных лидеров — пропа нет → CSS-оверрайд, см. `library-uikit`).
  - **токен** — exercised-токены переносим verbatim; шрифт не переносим; недостающие тона (hover) —
    инференс с пометкой либо Themer-маршрут.
  - **раскладка** — фикс-ширина макета → floor (ниже); интринзик-переполнения — канонами registry.
- **Компонент прежде дива:** raw-разметка инстанса (Tailwind-мимикрия структуры) = **спека раскладки и
  контента, не код для клона**. Мера — component-coverage (доля элементов в `g-*`-классах; эталон спайка ≈63%,
  остальное — раскладочная обвязка).

## Фикс-ширина макета и адаптив (floor)

Макет десктоп-фикс (адаптив не задан) → **объяви floor** (min-width сборки) в отчёте/README сборки; ниже
floor'а горизонтальный page-scroll допустим — это осознанная политика, не дефект. Гейт-находки ниже
объявленного floor'а — ожидаемые (не FP и не регрессия). Внутри floor'а действует канон «непереносящийся
тулбар → `overflow-x:auto`». Машиночитаемый `intendedMinWidth` в манифесте сценариев — кандидат S4/S6
(пока floor живёт в отчёте).

**Резиновая сетка из фикс-макета (owner-ruling S3).** Статичный макет переносится ТЯНУЩЕЙСЯ сеткой,
не фикс-ширинами колонок: контент-колонки `flex: 1 1 0` + `min-width: 0`, floor держит общий `min-width`.
Якоря растяжения — отступы: **слева = справа**, и/или горизонтальный отступ **совпадает с краем элемента
выше/ниже** (S3: правый край панели = правый край поиска в шапке, оба 20px от вьюпорта). Фикс-ширину из
макета оставляй только элементам с очевидным фикс-интринзиком (сайдбар 200, поле поиска 180); floor при
этом считай суммой: фикс-элементы + min контент-колонок + паддинги/гэпы (S3: 36px паддингов, забытых в
арифметике, дали дрейф эффективного floor'а против заявленного).

## Интерактивная семантика статичного макета (owner-ревью S3 r3, verified сборкой)

Статичный кадр несёт интеракционные контракты — читай их из СТРУКТУРЫ ноды, не только из видимого:

- **`.State-Highlighter` (скрытые дубли в строках списка) = модель ВЫБОРА строк**, не декорация одной
  выделенной: клик по строке выбирает её (деталь-панель показывает выбранную задачу), чекбокс в строке —
  **отдельный клик-слой** (completion; `stopPropagation`), не row-click. Выделенная в макете строка =
  начальный selected прототипа.
- **Узкий rounded-rect в зазоре панелей (≈2×32, вертикально по центру) = ресайз-хэндл сплита**, не декор:
  весь зазор — хэндл (`cursor: col-resize`, drag меняет доли панелей, кламп ~30–70%); пилюля красится
  токеном (в ките — `line-generic-accent`, тот же семантический, что бордер чекбоксов).
- **Вертикальная ось (рифма резиновой сетки):** список = скролл-зона (`flex: 1; min-height: 0;
  overflow-y: auto` — добирает пункты на высоком вьюпорте, скроллится на низком); терминальные зоны
  (композер/actions) прижаты к низу панели (`body { flex: 1 }` + полоса с `border-top`). Пустота под
  контентом при вертикальном ресайзе = промах переноса.
- **Дозаполнение прототипа (не противоречит verbatim):** verbatim — то, что макет ПОКАЗЫВАЕТ; зоны,
  которых он не показывает (свёрнутые выпадушки-секции, списки, обрезанные кадром), наполняй
  правдоподобным контентом того же вида — пустая выпадушка в живом прототипе = мёртвая зона. Пометка
  «не verbatim, дозаполнение» в конфликт-отчёте обязательна.

## Reply/редактор-зоны

Композер с wysiwyg-тулбаром в макете (письмо/комментарий/описание) → маршрут роутера «Редактор текста» =
`@gravity-ui/markdown-editor`, **ставь bundle целиком** — не реконструируй тулбар из `Button`+`TextArea`
(кит рисует компакт-тулбар, но канон «как по-настоящему» = настоящий редактор; его собственный тулбар
заменяет макетный — разница с макетом ожидаема, не конфликт). Высоту зоны ограничивай снаружи (координаты
из metadata). Обвязка и грабли (ToasterProvider, stickyToolbar, высота) — `library-markdown-editor`
(verified вкруткой: first-pass, Tier 0 + гейт чисто).

## Конфликт-отчёт (обязательный выход переноса)

Таблица: `# | класс (имя/раскладка/идиома/токен) | в макете | в коде/registry | решение + пометка`.
Плюс: список кит-протечек (флаг), заявленный режим канала, объявленный floor. Дисциплина как в verify:
**2 итерации на расхождение, дальше — класс в отчёт**, не полировка.

## Чего НЕ делать

- НЕ запрашивай design_context целого фрейма (не лезет) и НЕ читай все экземпляры повторяющегося блока.
- НЕ суди тона и высоты на глаз — пипетка/токен и координаты metadata.
- НЕ переноси шрифт кита; НЕ хардкодь hex вместо извлечённых токенов.
- НЕ используй инлайн-ассеты design_context как настоящие картинки (это плейсхолдеры компонента).
- НЕ угадывай unmapped/детачи — конфликт-отчёт.
- НЕ пиксель-дифф: сверка «глазами+гейт» (пиксель-перфект — ловушка; 2-итерации-правило).
- НЕ дублируй сюда версии пакетов — версии живут только в `registry.json`.
