# Каталог измерений конвенций (dimensions)

Декларация измерений, по которым «как принято» варьируется между сервисами: для каждого — *что задаёт · влияние на вывод · как обнаружить в коде/макетах · дефолт · поле схемы*. Это **семантика** слоя `service-profile`: схема (`schema/service-profile.schema.json`) задаёт **форму** (какие поля есть), а этот каталог — **смысл** (как каждое влияет на результат и как его распознать).

«Влияние» — не гипотезы: оно **измерено** прогоном `01-config-form` (один таск × baseline + 3 профиля → каждая ручка наблюдаемо сдвинула вывод) и `02-theming` (тема). Этот файл обобщает ту разовую таблицу в постоянный контракт.

## Кто это читает

- **Автосинтез профиля:** модель читает код/макеты сервиса + этот каталог → по колонке «как обнаружить» извлекает значения → заполняет поля профиля. Колонка «как обнаружить» — фактически черновик ТЗ на синтез. В **интервью-режиме** (`synthesis-interview.md`) та же логика применяется к описанию сервиса (что вывести / что спросить).
- **Сборщик** (greenfield): колонка «влияние» — карта, почему профиль двигает именно эти решения.
- **Brownfield** (`brownfield.md`): то же измерение, прочитанное «с другой стороны», — **поверхность влияния** (blast-radius правки). Поменял `density` → размеры везде; ребренд → одна тема → весь визуал.
- **Человек**, проектирующий профиль вручную.

## Правила синтеза

- **Слабый/неоднозначный сигнал → НЕ заполняй поле.** Омиссия = универсальный дефолт (precedence-правило слоя). Не угадывай ради полноты. Канон-формула для extraction: **«отсутствие кода ≠ политика»**.
- **Значение = универсальному дефолту → тоже омиссия** (или явная пометка «= дефолт»): default-equal-заполнения — шум (extraction-01).
- **Конфликт сигналов → бери доминирующий** (преобладает в большинстве экранов / самый свежий код), остальное — в заметку.
- **Marked choice → `preferred`.** Код/описание последовательно выбирает сторону «it depends»-развилки registry **вопреки универсальному склонению** — это сильнейший сигнал предпочтения сервиса, кодифицируй (extraction-01: пропущенный Slider-для-длительностей).
- **Аномалия ≠ конвенция.** Наблюдение, противоречащее универсальному дефолту/идиоме registry, — флаг-вопрос владельцу, не молчаливая запись в `conventions` (может быть причудой одной сборки).
- **Что не легло в структурное поле** → во freeform `conventions` (escape-hatch; оттуда же копит write-back-луп).
- Бренд/тема — особый случай: при синтезе ищи готовую тему (см. `theme` ниже), не реконструируй токены.

Режимы синтеза: из кода — `synthesis-extraction.md` (+ карта восстановимости измерений); диалогом — `synthesis-interview.md` (фаза 2 поверх extraction).

## Обзор

| Измерение | Что задаёт | Дефолт | Поле схемы |
|---|---|---|---|
| **theme** | тема + бренд | дефолтная тема Гравити (light) | `theme` |
| **density** | плотность | normal (`size="m"`) | `density.default` |
| **contentLayout** | раскладка контента | по центру + max-width | `contentLayout` |
| **formPolicy** | конвенции форм | uniform / FormRow / per-field | `formPolicy` |
| **approvedComponents** | сабсет + развилки | open | `approvedComponents` |
| **archetypes** | шаблоны страниц | по recipe registry | `archetypes` |
| **glossary** | доменный словарь | нейтральные имена | `glossary` |
| **referenceScreens** | эталоны сервиса | — | `referenceScreens` |
| **navigation** | каркас/ИА (главная, крошки + их размещение, футер страницы, back-аффорданс, паттерн нав) | по recipe | `navigation` |
| **interactionPolicy** | поведение (unsaved-гард, save-модель, optimistic) | явный save, без гарда | `interactionPolicy` |
| **feedbackPolicy** | глубина коммуникации (отзывчивость / пустые / загрузка / ошибки) | standard | `feedbackPolicy` |
| **componentStyling** | оформление (контент кнопок, политика view, тултипы) | text-кнопки | `componentStyling` |

> Структурно-поведенческие измерения (`navigation` / `interactionPolicy` / `feedbackPolicy` / `componentStyling` + `formPolicy.footerActions`) **промоутнуты в схему 2026-06-09** из `structural-conventions.md` после прогона 05 (доказали, что определяют характер). Их детальные «влияние / как обнаружить / граница» расписаны в `structural-conventions.md`; здесь они — первоклассные поля каталога. Профили из conventions → в эти поля размечаются по мере правок (follow-up).

## Измерения (детально)

### theme — тема и бренд
- **Что задаёт:** тема по умолчанию, переключатель, брендовая палитра.
- **Влияние на вывод:** цвет всего акцентного слоя — кнопки `view="action"`, `Switch`, `Slider`, selection, focus, ссылки, `text-brand-contrast`; светлая/тёмная база. *(02-theming: оранжевый vs синий — кнопка+Switch+Slider целиком.)*
- **Как обнаружить:** значение `theme=` у `ThemeProvider` → `base`; наличие переключателя (`useState<Theme>` + toggle) → `allowSwitch`; **готовая тема** — закоммиченный `*.theme.css` с блоками `.g-root_theme_*` или Themer `theme.json` → `themeConfig` (ссылайся на файл, не реконструируй); иначе россыпь `--g-*`-override / значение `--g-color-base-brand` → бренд-цвет (по возможности предложи сгенерировать тему через Themer).
- **Дефолт:** дефолтная тема Гравити (light), без бренда.
- **Поле:** `theme` (`base` / `allowSwitch` / `themeConfig` / `brandTokens`).

### density — плотность
- **Что задаёт:** размер контролов, шкалу отступов, типографику.
- **Влияние на вывод:** `size` у контролов; `gap`/`Box spacing`; выбор typography-вариантов. *(01: meridian `s` vs aurora `m`.)*
- **Как обнаружить:** преобладающий `size=` проп на контролах (`s`/`m`/`l`); типичные значения `gap` у `Flex` и `spacing` у `Box`; используемые `Text variant` (caption/body масштаб). ⚠️ Калибровка (extraction-01): `size="s"` → **compact** надёжно; **normal ↔ spacious** при `size="m"` различимы только «воздухом» (gap-шкала/padding/ширина полосы — spacious ≠ size l) — детект даёт кандидата, подтверждай интервью (confirm-класс). NB: в Patterns коллег это захардкожено как универсальная ось — у нас per-service.
- **Иконки в контролах:** размер иконки следует за `size` контрола — дефолт-шкала в registry `library-icons`
  (Button/инпуты xs12 s14 m16 l16 xl20; uikit не сайзит иконку сам). Сервис может переопределить шкалу под свою
  плотность; корректность слота (куда класть, что сайзить вручную) остаётся в registry. Профиль задаёт величины.
  Сюда же — **левый инсет старт-иконки** (зависит от размера иконки + контрола; правило «слева чуть больше верт.»;
  дефолт-шкала S/M:8 · L/XL:12 в `library-icons`, тюнится профилем вслед за размером иконки).
- **Дефолт:** normal (`size="m"`); иконки — registry-шкала.
- **Поле:** `density.default` (override icon-шкалы — кандидат на промоут в схему, пока через `conventions`).

### contentLayout — раскладка контента
- **Что задаёт:** ширину/выравнивание контента и адаптивность.
- **Влияние на вывод:** центрирование и max-width страницы vs full-width; брейкпоинты/мобильная раскладка. *(01: meridian full-width влево vs aurora центр 1120.)*
- **Как обнаружить:** обёртки контейнера — `justifyContent="center"` + `maxWidth`/`Container maxWidth` (→ centered-fixed / fluid-capped) vs их отсутствие (→ full-width); наличие `MobileProvider`/`useMobile`/media-брейкпоинтов → `adaptive.policy`; min-width-гейты → desktop-only.
- **Дефолт:** контент по центру с max-width.
- **Поле:** `contentLayout` (`width` / `maxWidth` / `align` / `adaptive`).

### formPolicy — конвенции форм
- **Что задаёт:** ширину полей, расположение подписи, размещение описаний, маркировку обязательных.
- **Влияние на вывод:** `FormRow direction`; ширины контролов (единые vs разные); `HelpMark` vs `FormRow.FieldDescription`; `required`-звёздочка vs form-level-заметка. *(01: meridian by-content/HelpMark/form-level vs aurora uniform/FieldDescription/per-field.)*
- **Как обнаружить:** преобладающий `FormRow direction` (`row`=слева / `column`=сверху); ширины контролов в формах — единые vs по контенту (+ конкретные бакеты для чисел/имён/описаний); частота `HelpMark`/`labelHelpPopover` против `FieldDescription`; `required` на полях против одной заметки «все обязательны».
- **Дефолт:** uniform (выровненная ширина) / `FormRow` / описания по смыслу / per-field.
- **Поле:** `formPolicy` (`fieldWidth` / `widthBuckets` / `formMaxWidth` / `labelPlacement` / `descriptionDefault` / `requiredMarking`).

### approvedComponents — сабсет и развилки
- **Что задаёт:** разрешённый набор компонентов и разрешение универсальных «it depends»-развилок.
- **Влияние на вывод:** какие пакеты/компоненты вообще используются; выбор на развилках (Slider↔NumberInput, uikit `Table`↔`@gravity-ui/table`). *(01: aurora Slider vs meridian NumberInput для «%».)*
- **Как обнаружить:** import-граф — какие `@gravity-ui/*` реально импортятся (сигнал `allow`); демонстративное отсутствие пакета (напр. нет `page-constructor` во внутреннем инструменте → кандидат в `deny`); повторяющийся выбор на развилках → `preferred`.
- **Дефолт:** open (любой компонент Гравити).
- **Поле:** `approvedComponents` (`policy` / `allow` / `deny` / `preferred`).

### archetypes — шаблоны страниц
- **Что задаёт:** любимые композиции экранов сервиса.
- **Влияние на вывод:** структуру новых экранов (каркас → секции → компоненты).
- **Как обнаружить:** повторяющиеся композиции (`AsideHeader` + фильтры + `Table`; `PageConstructor`-страница; форма-полоса с группами); структура `pages/`/`screens/`; общие layout-обёртки. Каждый архетип ссылается на recipe registry (`ref`), не заменяет его.
- **Дефолт:** собирать по recipe registry под тип задачи.
- **Поле:** `archetypes`.

### glossary — доменный словарь
- **Что задаёт:** как сервис называет сущности в UI (микрокопия).
- **Влияние на вывод:** лейблы, заголовки, тексты кнопок, плейсхолдеры.
- **Как обнаружить:** повторяющиеся доменные термины в лейблах/строках/i18n-ключах; имена сущностей в типах/моделях/API; устойчивые предпочтения формулировок.
- **Дефолт:** нейтральные имена из формулировки задачи.
- **Поле:** `glossary`.

### referenceScreens — эталонные экраны
- **Что задаёт:** какие экраны команда считает образцовыми.
- **Влияние на вывод:** прямого нет — это **источник** для синтеза остальных измерений и якорь «как у нас».
- **Как обнаружить:** экраны с «example»/«template» в имени; ключевые/самые свежие/часто-копируемые страницы продукта; то, на что ссылаются в ревью как на образец.
- **Дефолт:** —.
- **Поле:** `referenceScreens`.

### conventions — escape-hatch (не структурное измерение)
Freeform-правила, ещё не уложенные в поля выше. При синтезе сюда кладётся то, что распозналось, но не легло в структуру; write-back-луп копит сюда правки с прогонов, периодически дистиллируя в структурные измерения. Держать **только per-service** — универсальная корректность живёт в registry.
