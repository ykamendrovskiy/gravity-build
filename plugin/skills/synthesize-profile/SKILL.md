# Synthesize Profile — интервью-режим

Создай `service-profile.json` через **диалог** — per-service-слой «как принято именно у нас»
(тема/бренд, плотность, раскладка, форм-политика, навигация, сабсет компонентов, архетипы, словарь)
поверх универсального registry. Профиль потом потребляет скилл сборки (`create`).

## Когда применять

- Пользователь хочет задать / настроить / собрать **профиль сервиса** («как принято у нас») для
  сборок на Gravity UI, либо просит проинтервьюировать его про сервис.
- Профиль собирается с нуля через интервью (кода сервиса ещё нет). *(Синтез из кода/макетов —
  extraction/Figma-режимы, отдельные сиблинги; пока не упакованы.)*

## Как вести

1. **Прочитай плейбук и каталог** (рядом со скиллом): `service-profile/synthesis-interview.md` (как
   вести интервью) + `service-profile/dimensions.md` (что распознавать / что спрашивать) +
   `service-profile/schema/service-profile.schema.json` (контракт) + `service-profile/examples/*.json`
   (форма готового профиля). Веди строго по плейбуку.
2. **Интервью по плейбуку:** открой описанием сервиса → выведи из него то, что оно уже задаёт (по
   `dimensions.md`) → спрашивай **только дыры** (важные и не выводимые), дефолт — явной опцией →
   **омиссия = универсальный дефолт** (не выдумывай). «Заявлено, но ещё нет» (бренд без темы) →
   поле опусти + пометь TODO в `note`.
3. **Собери `service-profile.json`** по схеме; заполни `provenance` (интервью + какие вопросы
   задавались; что выведено / спрошено / оставлено на дефолте). Провалидируй по схеме.
4. **Запиши** профиль как `service-profile.json` в корень целевого проекта (туда смотрит скилл
   сборки) — или куда укажет пользователь. Покажи итог: что вывел / что спросил / что на дефолте.

## Принципы (из плейбука)

- Вывести-потом-подтвердить, не допрашивать; описание сервиса высокодоходно (~6/8 измерений).
- Омиссия = дефолт; профиль может быть тонким.
- Профиль задаёт только **открытые ручки** — не переопределяет корректность/идиому registry.

## Связь

- Профиль потребляет скилл сборки (`create`).
- Уточнение профиля из фидбэка прогонов — скилл `write-back`.
