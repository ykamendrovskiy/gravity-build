# pattern-landing — маркетинговая страница (агностичный спек вида)

> Тип: `pattern`. Что есть хорошая лендинг-страница, безотносительно сборки. Гравити-сборка — `recipe-landing`.

## Композиция (типовой порядок секций)

1. **Hero** — оффер одной фразой + 1-2 CTA (основной + вторичный).
2. **Фичи / ценность** — 3-6 пунктов (иконка + заголовок + текст).
3. **Социальное доказательство** — отзывы / логотипы клиентов.
4. **Тарифы** (если есть) — план-карточки, выделить рекомендуемый.
5. **FAQ** — частые возражения.
6. **Финальный CTA** — повтор основного действия.

## Состояния

Лендинг обычно **статичен** (контент известен на сборке) → полный контракт UI Stack не нужен, в основном **Ideal**.
Если секция тянет данные async (live-метрики, отзывы из API) — на ту секцию распространяется `interface-foundations`
(loading/empty/error по-блочно). Иначе — happy-path.

## Ручки сервиса (profile)

Тон/плотность, тёмный hero, набор секций, бренд-тема. Омиссия = дефолт.
